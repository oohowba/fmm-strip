# AGENTS.md — 給 AI 協作工具（codex 等）的工作守則

這個 repo 是一個純靜態網頁的原始碼。
主題：Open Mask（Invar 基材 FMM）鍍層移除藥水的研發實驗指南、進度與實驗記錄。

## 回覆與內容語言
- 一律使用**繁體中文**，口吻白話（讀者是老闆與現場人員，不是化學家）。
- 專有名詞第一次出現時，用一句白話解釋。

## 檔案結構
| 路徑 | 說明 |
|---|---|
| `site/index.html` | 網頁本體（純靜態、CSS 內嵌、零外部依賴）。研發指南、進度時間線、實驗記錄都在這裡改。 |
| `.github/workflows/pages.yml` | merge 進 main 後自動發佈到 GitHub Pages。**禁止修改**。 |

另外：`<head>` 裡的 `<meta name="robots" content="noindex, nofollow">` 是刻意設定（不讓搜尋引擎收錄），**不可移除**。

## 硬規則（違反即退件）
1. **字級 ≥15px**：內文 16px 起跳；次要文字/label/表頭/chip/hint 一律 ≥15px，嚴禁 13/14px。改完自己檢查：`grep -nE 'font-size:\s*1[0-4]px' site/index.html` 必須為零，換算後 <15px 的 em/rem 寫法也不行。
2. **手機優先 RWD**：≤480px 不能讓中文被壓窄逐字折行；表格在手機改卡片式或可橫向捲動。
3. **化學式排版**：用 `<sub>`/`<sup>` 或 Unicode 上下標，全檔一致。
4. **零外部依賴**：不引用任何 CDN、外部 JS、外部字型、外部 API。整個網站就是一個自足的 HTML 檔。
5. **不得把任何密鑰、token 寫進 repo**（`*.env` 已在 .gitignore）。
6. **內容安全**：實驗藥劑相關內容維持「研發實驗指南」定位——寫清楚安全防護（H₂O₂、MSA、有機溶劑的個人防護與廢液處理），不淡化風險。
7. **客戶保密**：這是公開 repo。網頁與任何檔案（含 commit 訊息、PR 說明）一律**不得出現客戶公司名稱、集團名、廠區代號**，統一寫「面板廠客戶（保密）」。實驗記錄裡也不要寫可以反推客戶身分的資訊。

## 工作流程
1. 開新分支改，改完 push 並開 Pull Request，說明改了什麼、為什麼。
2. PR 不必等別人審核，**自己確認過硬規則後即可直接合併**；合併進 main 後約 1 分鐘自動發佈上線。
3. **不要直接 push main**（一律走分支+PR，留下改動紀錄）。
4. 預覽：直接用瀏覽器打開 `site/index.html` 即可，不需要任何伺服器。

## 常見任務怎麼做
- **更新實驗進度/記錄**：改 `site/index.html` 的進度時間線區塊，已完成打 ✅、未完成 ⬜，附日期。實驗數據（批次、濃度、時間、失重等）可加表格，手機版記得卡片化或橫捲。
